Traitement des fichiers logs avec Power Query

Développement d’un processus Power Query pour nettoyer, restructurer et exploiter des fichiers logs texte non structurés enregistrés sur SharePoint.

Contexte

Lors de mon stage chez Orano DS, j’ai été amené à traiter des fichiers logs provenant de systèmes de suivi opérationnel. Ces fichiers sont au format texte, non structurés, et certaines entrées s’étendent sur plusieurs lignes.

Objectifs du projet

  • Agréger plusieurs fichiers logs provenant de SharePoint ;
  • Identifier et reconstituer les enregistrements multi-lignes ;
  • Extraire les champs clés : date, heure, type d’alerte, message ;
  • Transformer les données en table exploitable dans Power BI ;
  • Permettre une visualisation claire et un suivi des événements opérationnels.

Développement Power Query

Le traitement a été réalisé entièrement dans Power Query, incluant :

  • Connexion aux fichiers SharePoint et agrégation automatique ;
  • Filtrage et nettoyage des lignes superflues ;
  • Détection des débuts d’enregistrements et reconstitution des lignes multi-lignes ;
  • Extraction des champs essentiels via des transformations M ;
  • Création d’une table finale exploitable pour les rapports et visualisations.

Technologies utilisées

  • Power Query : traitement, nettoyage et transformation des fichiers logs
  • M : langage de requêtes Power Query pour manipulations avancées
  • Power BI : visualisation des logs transformés et analyse des alertes
  • SharePoint : stockage des fichiers source
